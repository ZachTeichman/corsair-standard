Security & Data Handling

How Your Resume Is Processed

  1. Your file is received over HTTPS and saved temporarily with a randomized identifier.
  2. The file is unzipped and its XML structure is read. It is never executed.
  3. Formatting rules run against the XML to produce a compliance report.
  4. An annotated copy is generated with Word-native comments at each issue location.
  5. Results are returned to your browser.
  6. Both files are scheduled for automatic deletion within 24 hours.

That is the complete lifecycle. Your resume does not go anywhere else.

What We Do Not Do

  • We do not read resume content for evaluation purposes.
  • We do not store resumes beyond the 24-hour deletion window.
  • We do not share files with third parties.
  • We do not use uploaded files for AI training.
  • We do not link file content to any identity.

File Security Measures

Randomized storage — files are stored under UUID-based identifiers, not your original filename.

No execution — DOCX files are parsed as ZIP/XML archives. We never open them in Word or any application that could execute embedded content.

Macro rejection — macro-enabled files (.docm) are rejected at upload before any processing occurs.

Size limits — uploads are capped at 10MB.

Rate limiting — upload frequency is limited per IP to prevent abuse.

HTTPS only — all connections require TLS encryption.

Automatic deletion — files are deleted within 24 hours regardless of whether you download them.

Microsoft Office Viewer

If the Office Viewer preview is enabled, your annotated DOCX is temporarily accessible at a public URL so Microsoft's viewer service can render it. This URL is tied to the temporary file and becomes inaccessible when the file is deleted. Microsoft's handling of files accessed through their viewer is governed by Microsoft's own privacy policy.
